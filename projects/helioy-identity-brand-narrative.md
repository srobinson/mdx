# Brand Narrative & Visual Language Guide

---

## 1. Origin Story

Every AI conversation starts from zero. Context loads, work happens, the session ends. Everything dissolves. The next session starts from zero again. Tokens spent rebuilding what was already known. Patterns rediscovered that were already mapped. Structure assembled that was already standing an hour ago.

This is the waste.

Stuart built the first version of context-matters because he got tired of re-explaining his own codebase to his own tools. Not a product decision. A frustration. He stored a few facts between sessions. Retrieved them at the start of the next one. The difference was immediate: fewer tokens, less repetition, better output. Not dramatically better. Measurably better.

That measurement mattered. It meant the problem was structural, not cosmetic. AI agents did not need better prompts. They needed memory.

So he built memory. Then he built structural awareness (fmm), because memory without understanding is just storage. Then coordination (nancy, helioy-bus), because agents that remember individually but cannot share are silos. Each tool addressed a specific failure mode. Each one reduced waste by a measurable amount.

The tools accumulated. They started reinforcing each other. Memory informed structure. Structure guided coordination. Coordination deposited better memory. A feedback loop formed. Not designed from the top down. Grown from the bottom up, shaped by what actually worked.

That growth pattern has a name in material science. When dissolved molecules settle onto a seed crystal, each one locking into position determined by the geometry of what came before, the structure self-organizes. Order emerges from accumulation. The crystal does not need a blueprint. It needs the right conditions.

Helioy is those conditions.

The name comes from HumanWork, the origin. Helios: sun, light, energy. The raw input that drives formation. Light enters a geometric structure. The structure refracts it, concentrates it, gives it direction. What was diffuse becomes precise. What was ephemeral becomes durable.

Light enters. Structure forms. Intelligence emerges.

## 2. Narrative Framework

The brand story follows three acts. These are not marketing constructs. They map to real stages in how Helioy's products work, how users adopt them, and how the visual language shifts across contexts.

### Act I: The Waste

**Product reality:** AI agents process tokens with no persistent state. Every session rebuilds from scratch. Structural questions require dozens of file reads. Context windows fill with redundant information. The cost is real and compounding.

**User experience:** The developer who has explained the same architecture to Claude for the fifth time this week. The team whose agents cannot share what they have learned. The organization burning budget on repeated discovery.

**Visual treatment:** Entropy. Particles in slow drift across dark space. Fading light trails that almost connect but dissolve before they reach each other. Warm amber dots losing luminosity as they scatter. The palette is muted: `surface-0` darkness with `text-tertiary` elements. Gold appears only as dying sparks.

This is not dramatic or alarming. It is quiet loss. The visual language should communicate waste through absence, not through chaos. Things are missing that should be there. Connections that should hold are failing silently.

**Emotional register:** Recognition. The viewer should see their own experience reflected. "Yes. This is what it feels like."

### Act II: The Crystallization

**Product reality:** Context-matters stores experience as geometric shapes. fmm indexes code structure for O(1) retrieval. Memory deposits accumulate. Each interaction adds value that subsequent interactions withdraw. The system begins to retain and refine.

**User experience:** The first time `cx_recall` returns exactly the right context without being told where to look. The moment `fmm_file_outline` replaces 15 file reads with one call. The session where the developer realizes the AI already knows the codebase architecture because it learned it three sessions ago.

**Visual treatment:** Formation. Scattered particles finding positions in an emerging lattice. Light trails that persist, layer, and brighten with each pass. The amber glow intensifies as structure coalesces. Geometry becomes visible: faceted surfaces, crystalline edges catching light. The palette warms: `gold-500` highlights appear on `surface-1` structures. Caustic light patterns begin to form on surrounding surfaces as the crystal refracts incoming energy.

The visual shift from Act I to Act II is not sudden. It is accretive. Each new particle that locks into the lattice makes the whole structure slightly brighter, slightly more defined. The animation should feel inevitable, like watching sugar dissolve in reverse.

**Emotional register:** Satisfaction. The pleasure of watching order emerge from noise. "This is working."

### Act III: The Choreography

**Product reality:** Multiple agents coordinate through helioy-bus without central scheduling. Nancy orchestrates task distribution based on accumulated context. Memory shapes behavior. The system anticipates needs based on patterns it has crystallized from prior experience. 5 fmm calls replace 30+ file reads because the structure is already known.

**User experience:** The builder steps back. Agents are communicating, sharing context, dividing work according to learned capabilities. The system moves with a fluidity that was not programmed but emerged from the accumulated interactions of Act II. Infrastructure becomes choreography.

**Visual treatment:** Resonance. Multiple light sources (agents) moving in coordinated but not rigid patterns. Their trajectories form emergent geometry when viewed from above. The crystal lattice is now a complete, luminous structure. Caustic patterns play across all surfaces. Individual facets catch and redirect light to each other. The palette reaches full expression: the complete warm mineral range from `surface-0` through `gold-500`, with `signal-success` sage accents on active connections. The structured light pattern system operates at full complexity.

Motion is the defining characteristic of Act III. Not speed. Precision. Elements move with the custom easing (`cubic-bezier(0.16, 1, 0.3, 1)`): decisive entry, controlled settle. Multiple staggered animations create the feeling of coordinated response without mechanical synchronization.

**Emotional register:** Aspiration. "I want my systems to work like this."

### Audience Mapping

| Audience | Primary Act | Entry Point | Key Visual |
|----------|-------------|-------------|------------|
| Vibe coders | Act I → Act II | Pain recognition | Before/after crystallization |
| Dev teams | Act II → Act III | Scaling frustration | Lattice growing as nodes join |
| Enterprise | Act III (full expression) | Operational efficiency | Choreography at scale |
| Future consumers | Act II (simplified) | Personal utility | Gentle persistent glow |

## 3. Visual Language Principles

### 3.1 Light as Protagonist

Light is the primary actor in Helioy's visual world. Not as decoration. As mechanism.

In the product, tokens are the raw input. In the visual language, light serves the same function. It enters the frame as undifferentiated energy (warm amber, `gold-400` to `gold-500`). It encounters structure (the crystalline geometry). It emerges transformed: concentrated, directed, multiplied through refraction.

Every visual composition should answer: where is the light coming from, what structure is it encountering, and how is it changed?

**Rules:**
- Light sources are warm. Always in the gold/amber range. Never cool, never white.
- Light interacts with structure. It does not float freely. When gold appears in a composition, something must be refracting, reflecting, or concentrating it.
- Light accumulates. Repeated interactions with the same structure make it brighter. This visualizes memory.
- Ambient light is `surface-0` darkness with slight amber warmth. The world is dark until Helioy's structures illuminate it.

### 3.2 Geometry as Grammar

Crystalline geometry is the structural grammar of the visual language. It governs form at every scale: the logo mark, the grid layouts, the data visualizations, the pattern system.

**Principles:**
- Facets, not curves. The visual world is built from planar surfaces meeting at angles. Curves appear only in caustic light patterns (which are themselves products of geometric refraction).
- Asymmetric balance. Perfect symmetry reads as static. Crystalline forms in nature are balanced but irregular. The logo uses this. Layouts use this (the design system's asymmetric column splits: 3/9, 4/8, 5/7).
- Internal structure. The most interesting property of a crystal is not its surface but its lattice. Visual compositions should suggest depth and internal organization. Translucent layers. Visible substructure. The sense of looking into something, not at it.
- Growth directionality. Crystals grow outward from a seed. Visual compositions can imply this: structure radiating from a focal point, complexity increasing toward the edges while the core remains dense and ordered.

### 3.3 Material Honesty

The visual language references real materials, not digital abstractions.

**What belongs:** Amber. Quartz. Warm stone. Oxidized metal. Matte charcoal surfaces with moments of crystalline reflection. These are materials formed through process and time. They carry associations with durability, craft, and accumulated geological force.

**What does not belong:** Gradient meshes. Glass morphism. Neon glow. Metallic chrome. AI sparkle effects. Abstract blob art. These are surface treatments with no material referent. They age with the trend cycle and communicate nothing about what Helioy builds.

The structured light patterns (caustics) are the bridge between material honesty and digital execution. They are generated from actual light physics, not illustrated. A viewer with knowledge of optics should recognize the patterns as physically plausible.

### 3.4 Density With Depth

Helioy's audience reads code. They handle information density. The visual language respects this by delivering density without clutter.

**How:**
- Layered composition. Multiple information layers at different depths. Primary content at the surface. Supporting structure visible beneath.
- Progressive revelation. Static compositions show the surface layer. Interaction (hover, scroll, zoom) reveals deeper layers. This mirrors how the product works: terse default output, verbose detail on request.
- Negative space as structure. The dark `surface-0` background is not empty. It is the unstructured space that makes crystallized structure visible. Like the darkness inside a geode that makes the crystals legible.

### 3.5 Motion as Narrative

Every animation tells a micro-story from the three-act framework.

**Formation animations** (Act II): Elements arriving and finding position. The custom easing provides the decisive entry (sharp attack) and the satisfying settle (long tail). Used for page loads, component mounts, data appearance.

**Choreography animations** (Act III): Multiple elements moving in coordinated response. Stagger patterns (0ms, 60ms, 100ms, 120ms) create ripple effects that suggest agents responding to shared context. Used for list reveals, dashboard updates, state transitions.

**Accumulation animations**: Elements that grow brighter, denser, or more structured with repeated interaction. Used for progress indicators, memory depth visualizations, usage metrics.

**What never moves:** The logo mark. The wordmark. Primary navigation. These are the stable geometry around which other elements choreograph. Stability in the frame reinforces the brand promise of persistence.

## 4. Metaphor Usage Guide

### The Crystallization Spectrum

The crystallization metaphor operates at different intensities depending on context. Too explicit and it becomes a gimmick. Too subtle and it disappears. The guide below maps intensity to context.

#### Level 1: Structural (Always On)

The metaphor is embedded in the design system itself. The user never encounters it as a metaphor. They experience it as how things look and move.

- Faceted geometry in the logo and pattern system
- Warm amber light on dark surfaces (light entering structure)
- Asymmetric layouts (crystalline rather than grid-perfect)
- Formation animations (elements finding position)
- Surface elevation through warmth shift (layers of accumulated depth)

These are invisible uses of the metaphor. They work whether or not the viewer knows anything about crystallization. A developer using the CLI feels the warmth and precision without parsing why.

#### Level 2: Suggestive (Marketing and Brand Moments)

The metaphor surfaces through visual composition and word choice, but is never named directly.

- Hero imagery showing structured light patterns (crystallization as visual atmosphere)
- Copy that uses accumulation language: "deposits," "compounds," "refines," "persists"
- Visual transitions from scattered to structured (the Act I → Act II shift)
- Product diagrams where data flows from diffuse to ordered
- The tagline "every token counts" (each token adds to the crystal)

At this level, a viewer who knows the metaphor will see it operating. A viewer who does not will simply find the visual language coherent and the copy precise. Both responses are correct.

#### Level 3: Explicit (Rare, High-Impact Moments)

The metaphor is named directly. Reserve for foundational brand moments.

- The origin story (Section 1 of this document)
- The "About" page on the website
- Conference keynote visuals
- Brand launch materials
- The guiding sentence: "Light enters. Structure forms. Intelligence emerges."

This level appears in contexts where the audience has already committed attention. A homepage visitor has not committed. A conference attendee has. Calibrate accordingly.

#### Level 4: Literal (Never)

The metaphor is applied as literal product language.

- ~~"Crystallize your workflow"~~
- ~~"Crystal-clear memory"~~
- ~~"Your data, crystallized"~~

This kills the metaphor by reducing it to a pun. The crystallization concept governs how we think and design. It does not appear in product copy as a label.

### Metaphor in Motion

Animation is where the metaphor is most powerful and most at risk of overuse.

**Use crystallization motion for:**
- First-time product experience (onboarding, empty states filling)
- State transitions that represent memory forming (saving, indexing, connecting)
- Hero animations on marketing surfaces
- Loading states where data is being structured

**Do not use crystallization motion for:**
- Every page transition (it becomes wallpaper)
- Error states (errors are not crystallization failures; they are system communication)
- Routine UI interactions (button clicks, menu opens, form submissions)
- Repeated user actions within a single session (the formation story is told once, not continuously)

### Metaphor Fatigue Test

If someone encounters Helioy across three touchpoints in one day (a tweet, the documentation, the product), they should experience the crystallization metaphor at three different intensities. The tweet might show a 3-second formation animation (Level 2). The documentation is structured light patterns in the background (Level 1). The product uses formation easing on component mounts (Level 1). At no point does the word "crystal" appear.

If all three touchpoints delivered Level 3 intensity, the metaphor would feel like a theme park. The gradient matters.

## 5. Tone Examples

The following examples demonstrate Helioy's voice across contexts. Each follows the voice principles from the brand strategy: say less, mean more. Show the mechanism. Respect the reader's time. Be specific. Warm authority.

### 5.1 Website Hero

> **Agents that remember.**
>
> Helioy gives AI agents persistent memory, structural awareness,
> and coordinated communication. Context compounds instead of
> evaporating. 5 calls replace 30 file reads.
>
> Every token counts.

*Notes: Lead with the value proposition in three words. Second line shows the mechanism (what the products do). Third line provides a specific metric. Tagline anchors. No adjectives. No "revolutionary." No "AI-powered."*

### 5.2 Product Description (context-matters)

> Structured context store for AI agents.
>
> context-matters persists facts, decisions, and patterns between sessions
> using geometric similarity, not keyword matching. Store once, recall by
> resonance. Retrieval improves as the store grows.
>
> `cx_recall` searches by meaning. `cx_deposit` saves what matters.
> Five MCP tools. One persistent memory.

*Notes: Noun phrase opening identifies what it is. Second paragraph shows the mechanism (geometric similarity, resonance). Third paragraph is concrete: tool names, counts. The reader knows exactly what they are getting.*

### 5.3 Error Message

> **Connection lost to helioy-bus.**
>
> Agents can still operate independently. Shared context
> will sync when the connection restores.
>
> Check: `hb status` or restart with `hb reconnect`

*Notes: State the problem. State the impact (reassuring: agents still work). Provide actionable next steps. No apology. No exclamation marks. No "Oops!" The tone is a competent colleague reporting a status change.*

### 5.4 Changelog Entry

> **context-matters 0.9.0**
>
> Geometric retrieval now supports scope-chain resolution. A query
> scoped to `project:helioy/repo:nancyr` also searches `project:helioy`
> and `global`, returning the most relevant entries regardless of where
> they were stored.
>
> Token cost for recall dropped 40% through smarter scope pruning.
>
> Breaking: `cx_recall` now requires `scope` parameter. Previously
> defaulted to global. Migration: add `scope: "global"` to existing calls.

*Notes: Version number and name. What changed, with a concrete example. Measurable improvement (40%). Breaking change stated plainly with migration path. No celebration. No "exciting new feature." The improvement speaks for itself.*

### 5.5 Tweet / Short-Form Social

> Your AI rebuilt your project context 47 times this week.
>
> Mine remembered it from Monday.
>
> helioy.dev

*Notes: Specific number (47 is more believable than 50). Contrast between waste and memory. URL. That is the entire tweet. No hashtags. No emoji. No "check it out!" The reader either recognizes the pain or does not.*

### 5.6 Documentation Introduction

> # Getting Started
>
> Helioy is a set of MCP servers that give AI agents persistent
> memory and structural awareness. This guide covers installation,
> first configuration, and a working example.
>
> **Prerequisites:** Node.js 18+, an MCP-compatible client (Claude Code,
> Cursor, or similar), 10 minutes.
>
> **What you will build:** A context store that persists facts between
> sessions and a code index that answers structural questions without
> reading files.

*Notes: First sentence defines what Helioy is. Second sentence defines what this page covers. Prerequisites are specific (version, time estimate). "What you will build" sets a concrete expectation. No "Welcome to Helioy!" No "In this comprehensive guide, we will walk you through..."*

### 5.7 Empty State (Product UI)

> **No context stored yet.**
>
> This store is empty. Run `cx_deposit` to save your first entry,
> or `cx_recall` with a query to search ancestor scopes.
>
> Stores grow more useful over time. Start small.

*Notes: State the fact. Provide two actionable paths. The final line is the closest the product voice gets to encouragement: a quiet acknowledgment that starting is the hardest part. No sad-face illustrations. No "It's lonely in here!" The empty state is the visual equivalent of Act I: open space waiting for structure.*
